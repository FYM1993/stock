```
震荡市场：

  第二优先级：情绪冰点日

```

### 2. 买入逻辑分支（Phase 4）
```python
if market_regime == OSCILLATE_ROTATION:
    # 前龙头修复套利
    if is_ice_point and is_previous_leader:
        买入前龙头（套利逻辑）
    
elif market_regime == OSCILLATE_SECTOR:
    # 重仓板块最强
    板块起爆、龙头回踩、适度追涨
    
elif market_regime == STRONG:
    # 全仓最强龙头

```

### 3. 卖出逻辑分支（Phase 5）
```python
if market_regime == OSCILLATE_ROTATION:

        立即卖出
    
elif market_regime == OSCILLATE_SECTOR:
    # 见好就收
    if profit > 5% or break_ma5:
        卖出
    
elif market_regime == STRONG:
    # 追求连续性
    if 板块高潮 or break_ma5 or profit > 30%:
```

```python
记录：leader_peak_date = 今天
```

```python
def _check_sector_continuity(self, date, window=3):
    """
    检查过去3天的龙头历史：
    - 如果同一板块龙头连续出现 ≥2次 → 有板块效应
    - 如果每天都是不同板块 → 板块轮动
    """
```

逻辑：
1. 查看过去3天的龙头股及其所属板块
2. 统计每个板块出现次数
3. 如果某板块出现≥2次 → 有板块效应

### 📖 策略文档核心理念

根据《龙头低吸.md》：

#### 强势市场（牛市）
- 特点：指数上升、主线清晰、龙头明确
- 逻辑：**重仓参与主升**，做最强板块最强个股
- 统计区间起点：**指数爆发日**（第一优先级）
- 买点：冰点低吸、板块分歧、弱为强、回流
- 卖点：板块高潮、跌破五日线

#### 震荡市场
- 特点：指数稳定、无主线或多板块轮动

- ⚠️ 快速轮动期：不操作

#### 弱势市场（熊市）
- 特点：指数空头排列、无主线、闪崩频发
- 逻辑：**空仓休息或轻仓试错**
- 买点：超跌+止跌、冰点反弹
- 卖点：次日兑现、不黏股

```
Day 1: 择时触发 → 区间 = 1天  (start → today)
Day 2: 无新择时 → 区间 = 2天  (start → today)

Day 31: 新择时触发 → 重置，区间 = 1天 (重新开始)
```

**龙头定义**：
> "从择时信号开始到现在，累计涨幅最高的就是龙头"

### 技术亮点

**动态板块聚类算法**：
```python
1. 计算区间内所有股票的日收益率
2. 生成相关性矩阵（corr_matrix）
3. 贪心聚类：相关性>0.7的股票归为一组
4. 筛选：板块成员3-20只，平均涨幅>15%
```

**龙头3维度评分**：
```python
- 累计涨幅：整个周期涨幅（50分权重）
- 提前启动：前1/3区间涨幅（30分权重）
- 量能极限：后半段/前半段量比（20分权重）
```

### 新增方法

- `_identify_hot_sectors()` - 动态识别热点板块
- `_calculate_leader_score()` - 3维度龙头评分
- 改进 `_timing_signal()` - 3种择时方式
- 改进 `_update_leader_candidates()` - 基于板块选龙头

## 已知问题与限制

### v2.0 当前版本

1. **官方数据更新延迟**
   - 问题：官方数据更新到 2024-05-22，可能不是最新
   - 影响：需要补充 ~630 天的数据
   - 解决：使用独立脚本补充

2. **AKShare 限流风险**
   - 问题：大量请求可能被临时限流
   - 影响：下载速度降低或失败
   - 解决：启用活跃度过滤、增加延迟、分批下载

3. **存储空间需求**
   - 问题：完整数据需要 ~5-8 GB 空间
   - 影响：磁盘空间不足时下载失败
   - 解决：清理旧数据、只下载必要范围
