# v18 设计：使用静态行业分类替代动态板块识别

## 核心问题

v17失败的根本原因：**板块识别失败率38.6%**

- 动态相关性聚类过于复杂，数据要求高
- 统计区间短时数据不足，导致识别失败
- 识别失败后fallback到全市场涨幅，选不到真龙头
- 导致强势市场也无法全仓买入龙头，错失主升浪

## v18 改进方案

### 1. 使用AKShare静态行业分类

**API**: `ak.stock_individual_info_em(symbol="股票代码")`

**优势**:
- 稳定可靠，每只股票都有明确的行业归属
- 无需动态计算，不受统计区间限制
- 东方财富的分类与同花顺基本一致

**示例**:
```python
# 新易盛 300502
info = ak.stock_individual_info_em(symbol="300502")
# 返回: 行业 = "通信设备"
```

### 2. 修改板块识别逻辑

#### 2.1 预加载行业映射

**启动时一次性获取所有股票的行业信息，缓存到本地**:

```python
# data/stock_industry_mapping.json
{
  "300502.SZ": "通信设备",
  "600589.SH": "通信设备",
  "300502.SZ": "光学光电子",
  ...
}
```

**更新策略**:
- 每天首次运行时更新缓存
- 若API失败，使用本地缓存

#### 2.2 动态热点行业识别

**逻辑**:
1. 遍历统计区间内的所有候选股（涨幅top 50-100）
2. 根据行业映射，统计每个行业的股票数和平均涨幅
3. 筛选热点行业：
   - 平均涨幅 > 15%（牛市）或 > 10%（震荡）
   - 成员数 >= 3只
   - 涨停/大涨股数 >= 2只

**示例**:
```
统计区间: 2025-06-19 → 2025-07-01 (情绪冰点)

候选股（涨幅top 50）:
- 300502.SZ 新易盛  +45.2%  行业: 通信设备
- 600589.SH 广东榕泰 +38.1%  行业: 化工
- 002396.SZ 星网锐捷 +32.5%  行业: 通信设备
- 300308.SZ 中际旭创 +31.8%  行业: 通信设备
...

行业统计:
- 通信设备: 12只, 平均涨幅 +28.3%, 涨停数 5
- 化工: 3只, 平均涨幅 +18.2%, 涨停数 1
- 光学光电子: 8只, 平均涨幅 +25.1%, 涨停数 3

热点行业识别结果:
✅ 通信设备 (CPO概念核心)
✅ 光学光电子
```

### 4. 市场状态判定简化

v17的4种状态太复杂，**简化为3种**:

| 市场状态 | 判定条件 | 仓位策略 | 买点逻辑 |
|---------|---------|---------|---------|
| **强势** | 大盘MA5/MA20/MA60多头<br>涨停数>跌停数2倍<br>热点行业持续3天以上 | 1只，100% | 追涨龙头<br>板块起爆分歧 |
| **震荡** | 大盘在MA20±5%之间震荡<br>涨跌停数相当<br>有热点行业（但不持续） | 2只，各40% | 冰点低吸<br>回踩买入<br>前龙二波 |
| **弱势** | 大盘跌破MA20<br>跌停数>涨停数<br>无热点行业 | 0只，空仓 | 不参与 |

**关键改进**:
- 去掉"震荡-轮动"vs"震荡-板块效应"的复杂区分
- 只要有热点行业，就按"震荡"处理（2只40%）
- 强势的判定更严格（需要热点持续3天）

## 风险点

1. **AKShare API稳定性**
   - 缓解: 本地缓存兜底
   
2. **行业分类粗糙**
   - 例如"通信设备"包含CPO + 其他
   - 缓解: 结合涨幅和量能二次筛选

3. **强势判定过严**
   - 可能漏掉小周期机会
   - 缓解: 震荡仓位40%，不算太低
