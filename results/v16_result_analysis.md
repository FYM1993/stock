## ❌ v16问题分析

### 问题1：量能基准期选择不对
**当前逻辑**：
```python
统计区间：2025-06-05 → 2025-06-17（12天）
历史基准：2025-04-25 → 2025-06-05（40天）
```

**问题**：
- 新易盛5月中就开始启动放量
- 历史基准（4/25-6/5）已经包含了启动期
- 基准量能被拉高，导致量比只有1.07倍

**示例**：
```
4月（平静期）：日均成交80
5月（启动期）：日均成交200  ← 已经放量
6月（加速期）：日均成交250

当前计算：
历史基准（4/25-6/5包含5月）：平均150
当前8天（6/5-6/17）：平均250
量比：250/150 = 1.67倍（勉强10分）

但实际应该：
历史基准（4月）：平均80
当前8天：平均250
量比：250/80 = 3.1倍（30分）✅
```

## 💡 v17改进方案

### 方案1：修正量能基准期（必须）
```python
# v16（错误）
hist_start = start_date - 40天
hist_end = start_date  # 紧邻统计区间

# v17（正确）
# 往前推更远，避开启动期
hist_end = start_date - 5天  # 留5天缓冲
hist_start = hist_end - 30天  # 取之前30天作为基准

# 或者：直接用统计区间之前60-90天
hist_start = start_date - 90天
hist_end = start_date - 60天
```

**效果**：
- 基准期完全在启动之前
- 量比会显著提高（1.07 → 2.5-3倍）
