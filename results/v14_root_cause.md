```python

### 问题剖析

**统计窗口只有12天（<15天）**：
- 代码设置：`lookback_days = 15`
- 实际统计：`tracking_start_date = 2025-03-06`, `end_date = 2025-03-18`
- 结果：只有12个交易日数据

**为什么12天不够？**
1. **龙头需要发酵时间**：启动 → 突破 → 加速 → 高潮 → 回落，完整周期>30天
2. **3月18日是初期**：长飞刚启动，涨幅5.3%属于正常起步
3. **量能还未爆发**：初期量能温和（1.01倍），爆量通常在加速期（高潮前）
4. **评分标准太高**：要求10%涨幅才有分，但12天5%已经很强了（日均0.4%）

```

## 💡 解决方案

### 方案1：延长统计窗口 ⭐推荐
```python
# 当前
lookback_days = 15

# 修改为
lookback_days = 30  # 或者40、60
```

**优点**：
- 简单直接
- 能覆盖完整龙头周期
- 更准确反映"整个行情结束它涨幅最高"的定义

**缺点**：
- 窗口太长可能包含多个行情周期
- 但可以通过动态窗口解决（见方案3）

### 方案2：降低评分阈值
```python
# 累计涨幅调整
> 30%: 50分  # 从50%降到30%
> 20%: 40分  # 从30%降到20%
> 10%: 30分  # 从20%降到10%
>  5%: 20分  # 新增：5%就有分

# 提前启动调整
> 15%: 30分  # 从20%降到15%
> 10%: 25分  # 从15%降到10%
>  5%: 20分  # 从10%降到5%
>  3%: 10分  # 新增：3%就有分
```

**优点**：
- 对短期行情更敏感
- 不用改窗口长度

**缺点**：
- 可能误把短期炒作当龙头
- 评分标准不够严格

### 方案3：动态统计窗口 ⭐⭐最佳
```python

if market_regime == 'strong':  # 牛市

elif market_regime == 'weak':  # 熊市
    lookback_days = 20  # 熊市反弹快，用20天
else:  # 震荡
    lookback_days = 30  # 震荡适中，用30天
```

**优点**：
- 自适应市场环境
- 牛市捕捉大行情，熊市捕捉快速反弹
- 更符合实际

**缺点**：
- 实现稍复杂
- 需要测试各环境下的最优值
