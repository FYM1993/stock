# v14 重大改进：并行买点检测

## 🎯 核心改进

### 1. 买点检测从串行改为并行
**之前（v13）**：
```python
for symbol in candidates:
    if 冰点:
        return '冰点'  # 遇到第一个满足的就返回
    if 板块分歧:
        return '板块分歧'
    # 后面的买点检测不到了
```

**现在（v14）**：
```python
# 第1步：并行检测所有候选股的所有买点
for symbol in candidates:
    signals = check_all_buy_signals(symbol)  # 检测所有买点
    # signals = [('ice_point', 100), ('weak_as_strong', 70), ...]
    
# 第2步：按优先级排序
signals.sort(key=优先级+龙头分)

# 第3步：依次买入
for signal in signals:
    buy(signal)
    if 满仓: break
```

### 2. 删除冗余的"板块分歧低吸"
- **删除理由**：与"板块起爆分歧"逻辑重叠，且看的是全市场维度（不精准）
- **保留**：情绪冰点（市场维度）+ 板块起爆分歧（板块维度）

### 3. 买点优先级评分系统

| 买点 | 优先级分数 | 说明 |
|------|----------|------|
| 情绪冰点 | 100分 | 市场极度弱势，反弹机会（市场维度）|
| 板块起爆分歧 | 90分 | 板块首次涨停潮后分歧（板块维度）|
| 弱为强之胆 | 70分 | 板块内部轮动 |
| 技术面低吸 | 60分 | 缩量回踩5日线 |
| 强势突破 | 50分 | 牛市追涨 |

**排序逻辑**：
1. 优先级分数（高优先）
2. 龙头评分（Top1=100分，Top2=95分...）

## 📊 解决的问题

### 问题：长飞光纤在候选池但没买入
**原因**：
- v13串行遍历，前面的股票买了就满仓
- 长飞排第7，轮不到检测

**v14解决**：
- 先检测所有候选股的所有买点
- 收集所有信号，按优先级排序
- **长飞如果有高优先级买点（如板块起爆分歧90分），会排到前面买入**

## 🔍 预期效果

1. ✅ **龙头不会被遗漏**：即使龙头分数不是第1，只要有高优先级买点，也会优先买入
2. ✅ **买点更科学**：删除冗余逻辑，保留核心买点
3. ✅ **可扩展性强**：新增买点只需修改`_check_all_buy_signals`
