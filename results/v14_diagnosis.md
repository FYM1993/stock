## 🔍 问题根因分析

### 现象
查看日志发现：
1. 板块识别工作正常（识别到15个、28个、45个板块）
2. 候选池只有10只
3. Top3分数示例：`002105.SZ(90分), 601020.SH(80分), 300611.SZ(75分)`
4. **真龙头不在候选池里**

### 可能原因

#### 1. 板块识别太松散
- 相关性>0.7就聚为一个板块
- 可能太松了，识别出太多无关的"伪热点"
- 真正的热点板块被淹没

#### 2. 3维度评分有问题
当前评分：
```python
累计涨幅：40分（整个周期）
提前启动：40分（前1/3周期）
极限量：  20分（后半vs前半成交量比）
```

**问题猜测**：
- 长飞光纤如果在板块里，但涨幅可能不是最高（因为启动早，后期横盘）
- "提前启动"的评分可能不够敏感
- "极限量"的计算可能不准确

#### 3. 统计周期固定15天
```python
lookback_days = 15
self.tracking_start_date = (current_date - pd.Timedelta(days=15)).strftime('%Y-%m-%d')
```

**问题**：
- 真龙头可能在15天前就启动了
- 15天窗口内，龙头可能已经涨了很多，不再符合"提前启动"特征
- 反而是后启动的跟风股在15天窗口内涨幅更大

## 💡 改进方案

### 方案1：优化板块识别（优先）
- 提高相关性阈值：0.7 → 0.8
- 提高涨幅阈值：动态8% → 12%
- 减少识别出的板块数，聚焦真热点

### 方案2：调整评分权重
```python
# 当前
累计涨幅：40分
提前启动：40分
极限量：  20分

# 建议
累计涨幅：50分（最重要：整个行情它最高）
提前启动：30分（重要但不是最重要）
极限量：  20分
```

或者干脆：
```python
# 更激进
累计涨幅：60分
极限量：  40分（与自己历史对比）
提前启动：不评分，改为必要条件（必须在前1/3启动）
```

### 方案3：扩大统计周期
```python
# 当前：固定15天
lookback_days = 15

# 建议：动态20-30天
lookback_days = 30  # 或根据市场环境动态调整
```

### 方案4：直接用区间涨幅排序
**最简单粗暴的方案**：
- 不用复杂的3维度评分
- 直接用统计区间内的累计涨幅排序
- Top10就是涨幅最高的10只
- 理论依据：龙头定义就是"整个行情结束它涨幅最高"
