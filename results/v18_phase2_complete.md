## 🔍 关键改进点

### 1. 为什么要合并"震荡-板块效应"和"震荡-轮动"？

**v17的问题**:
```python
# 板块连续性检查
for i in range(3):
    check_date = dates[i]
    if check_date in self.leader_history:
        sector = self.leader_history[check_date]['sector']
        ...

# 问题：
# 1. leader_history 经常为空（板块识别失败）
# 2. 即使有记录，sector也可能是None
# 3. 导致误判为"轮动"
```

**v18的简化**:
```python
# 直接判断
if has_hot_sectors:  # 只要识别到热点
    if meets_strong_conditions:
        return STRONG  # 强势100%
    else:
        return OSCILLATE  # 震荡40%
else:
    return WEAK  # 弱势空仓
```

### 2. 为什么"震荡"统一40%？

**理由**:
1. v17的20%太保守，错失机会
2. 40%是合理的中等仓位
3. 即使"轮动"也有短期套利机会
4. 简化决策，减少误判

### 3. 如何判断"强势"的"热点持续"？

```python
# 检查过去3天是否有龙头记录
continuous_count = 0
for i in range(1, 4):
    check_date = (current_date - timedelta(days=i)).strftime('%Y-%m-%d')
    if check_date in self.leader_history:
        continuous_count += 1

# 3天中有2天以上有龙头 → 热点持续
hot_sector_continuous = (continuous_count >= 2)
```

**优势**:
- 不依赖板块连续性（不检查sector是否相同）
- 只检查是否有龙头记录（更宽松）
- 避免过于严格的判定
