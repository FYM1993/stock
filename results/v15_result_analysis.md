## 🔍 根因分析

### 问题1：情绪冰点判断太敏感
**现状**：
- 5-6月是牛市，但期间有多次短期回调/分歧
- 每次小回调都触发"情绪冰点"
- 导致统计区间频繁重置

**为什么5-6月牛市会有"情绪冰点"？**
- 牛市上涨过程中，会有分歧日（涨停股减少）
- 市场轮动时，前期龙头回调，新龙头启动
- 短期调整时，上涨家数暂时减少

**本质**：
- 我们用的"情绪冰点"是**震荡/熊市**的择时信号
- 不应该在**牛市上涨途中**频繁触发
- 牛市应该用"指数择时"（大行情启动时触发一次，然后一直统计）

### 问题2：缺少"前龙见顶日"判断
**当前实现**：
```python
# 方式3：每30天强制重置
if days_diff >= 30:
    return True
```

**问题**：
- 30天固定重置，没有考虑市场环境
- 牛市行情可能持续2-3个月，不应该30天重置
- 应该检测"前龙是否见顶"，而不是固定天数

## 💡 改进方案

### 方案1：择时信号分市场环境（推荐）

```python
def _timing_signal(self, date: str) -> bool:
    market_regime = self._identify_market_regime(date)
    
    # 牛市：只用指数择时（大行情启动时触发一次）
    if market_regime == 'strong':
        # 检测指数大突破（涨幅>3% 且 量比>2）
        if 指数突破():
            return True
        # 牛市期间不用情绪冰点！
        # 不强制30天重置，让统计区间覆盖整个牛市周期
        return False
    
    # 震荡/熊市：用情绪冰点 + 周期重置
    else:
        if 情绪冰点():
            return True
        if days_diff >= 30:  # 震荡市30天重置
            return True
        return False
```

**效果**：
- 5-6月牛市：只在启动时触发一次，统计区间覆盖60-90天
- 震荡市：保持现有逻辑，5-10天重置

### 方案2：情绪冰点判断优化

```python
# 当前：情绪冰点 = 涨停数 < 20%分位
# 问题：牛市分歧日也会触发

# 改进：情绪冰点 + 市场下跌
if self.emotion_manager.is_ice_point(date) and 指数下跌():
    return True
```

**效果**：
- 牛市分歧日（指数不跌）：不触发
- 真正恐慌（指数跌 + 涨停少）：触发

### 方案3：动态周期重置

```python
# 当前：固定30天
if days_diff >= 30:
    return True

# 改进：根据市场环境
if market_regime == 'strong':
    max_days = 90  # 牛市允许更长周期
elif market_regime == 'weak':
    max_days = 20  # 熊市快速重置
else:
    max_days = 30  # 震荡适中

if days_diff >= max_days:
    return True
```
