## 🎯 根本原因

### 1. 统计区间只有1天的原因
```
6/19 触发情绪冰点
   ↓
tracking_start_date = 2025-06-19
   ↓
6/20 执行决策时，区间 = 6/19 → 6/20 = 1天！
```

### 2. 1天统计导致的问题
- **板块识别失败**：1天涨幅无法识别板块，成功0只
- **全市场选股**：退回到`pred_score`（历史累计涨幅）
- **选到了错票**：600589、002323在历史上涨幅高，但不是当前周期的龙头

### 3. 为什么不是未来涨幅的龙头？
因为`pred_score`用的是**历史累计涨幅**，不是**未来预测**！

```python
# 当前代码逻辑
if not hot_stocks:
    # 板块识别失败，用全市场pred_score
    candidates = self.pred_score.nlargest(top_n)
```

**`pred_score`是什么？**
- 策略初始化时计算的**历史统计数据**
- 不会动态更新！
- 所以6/20选的是"历史上涨幅高的票"，不是"未来会涨的票"

## 💡 为什么会统计区间只有1天？

### 择时信号触发逻辑
```python
# 情绪冰点判定（第6次修正后）
ice_thresh = self.data_provider.limit_up_count.quantile(0.15)
if limit_up_cnt <= ice_thresh:
    logger.info(f"🧊 情绪冰点: 涨停数={limit_up_cnt} ≤ 阈值{ice_thresh:.0f}")
    return True
```

**问题**：`quantile(0.15)`是15分位数，很容易触发！

### 6月冰点触发记录
```
6/3冰点  → 统计区间从6/3开始
6/4      → 区间=1天
6/5      → 区间=2天
...
6/19冰点 → 重置！统计区间从6/19开始
6/20     → 区间=1天（买入了600589、002323）
```

**冰点太频繁 → 区间永远拉不长 → 板块识别失败 → 选错票！**
